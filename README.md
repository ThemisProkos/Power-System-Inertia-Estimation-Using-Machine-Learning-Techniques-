# Thesis: Machine Learning-Based Estimation of Power System Inertia Constants

## Overview
The energy transition of recent years has been characterized by the rapid increase in the share of Renewable Energy Sources (RES) in the global energy mix, replacing conventional thermal units based on fossil fuels. Although the growing penetration of RES significantly contributes to the decarbonization of the power sector, it has also introduced new technical challenges in the operation of modern power systems. One of the most significant challenges arising from this transition is the reduction of power system inertia.

Conventional synchronous generators inherently provide physical inertia to the grid through their rotating masses. In contrast, RES units, which are connected to the grid via power electronic converters, do not directly contribute to inertia, leading to a considerable reduction in the total stored kinetic energy of the system. Limited inertia decreases the ability of the power system to absorb sudden power imbalances, making frequency control more challenging and increasing the likelihood of instability phenomena.

Therefore, the real-time estimation and monitoring of system inertia constitute key factors for ensuring frequency stability and supporting timely decision-making by system operators. This thesis develops a methodology for estimating inertia levels through the analysis of power system dynamic responses. The proposed approach relies on the application of probing signals generated by energy storage systems and on the utilization of machine learning techniques.

## Power System Scenarios

  ### Scenario 1: 2-Area System with Mixed Conventional Generation
  - **Area 1**: One single reheat steam turbine (linear model) + One non-reheat steam turbine (parallel connection)
  - **Area 2**: One non-reheat steam turbine + One hydro turbine (linear model)
  - **Purpose**: Baseline scenario with conventional generation mix
  
  ### Scenario 2: 2-Area System with Hydro Dominance
  - **Area 1**: Two hydro power turbines (parallel connection)
  - **Area 2**: One single reheat steam turbine + One non-reheat steam turbine
  - **Purpose**: Study hydro-dominated system dynamics
  
  ### Scenario 3: 2-Area System with Symmetric Configuration
  - **Area 1**: One single reheat steam turbine + One hydro turbine
  - **Area 2**: One single reheat steam turbine + One hydro turbine
  - **Purpose**: Analyze symmetric system behavior
  
  ### Scenario 4: 3-Area Interconnected System with Diverse Generation
  - **Area 1**: One single reheat steam turbine + One non-reheat steam turbine
  - **Area 2**: One hydro turbine + One non-reheat steam turbine
  - **Area 3**: One single reheat steam turbine + One biomass power plant turbine + One gas turbine
  - **Purpose**: Complex interconnected system with diverse generation technologies

## Methodology
  The proposed methodology follows these steps:
  
  1. **System Modeling**: Power systems are modeled in MATLAB/Simulink environment with detailed governor-turbine dynamics
  2. **Data Generation**: Monte Carlo simulations generate 1000 different operating scenarios with varying inertia constants
  3. **Feature Extraction**: Statistical indicators (standard deviation, max/min values, etc.) are extracted from frequency and power signals
  4. **Machine Learning Training**: Five ML algorithms are trained on the extracted features
  5. **Performance Evaluation**: Algorithms are compared based on accuracy metrics (mean percentage error, R¬≤ score)

## Data Generation

  A comprehensive dataset was generated through extensive Monte Carlo simulations. For each of the four power system scenarios, **1000 distinct simulations** were conducted, resulting in a total of **4000   simulation cases** across all scenarios.
  
  ### Simulation Parameters
  - **Inertia Constant Variation**: In each simulation, the combination of inertia constants (H‚ÇÅ, H‚ÇÇ) was varied within the realistic range of **[3, 8] seconds**. This range represents actual operating conditions in modern power systems where inertia values naturally fluctuate.
  - **Random Sampling**: The (H‚ÇÅ, H‚ÇÇ) pairs were randomly sampled from uniform distributions across this range to ensure comprehensive coverage of the parameter space.
  
  ### Measured Responses
  For each simulation, the following dynamic responses were recorded:
  1. **Œîf‚ÇÅ**: Frequency deviation in Area 1
  2. **Œîf‚ÇÇ**: Frequency deviation in Area 2  
  3. **ŒîP_tie**: Tie-line power flow deviation between the interconnected areas
  
  These measurements capture the system's transient behavior following disturbances or probing signals.
  
  ### Dataset Structure
  Each simulation produces:
  - **Input**: A unique inertia constant pair (H‚ÇÅ, H‚ÇÇ)
  - **Output**: Three corresponding time series (Œîf‚ÇÅ(t), Œîf‚ÇÇ(t), ŒîP_tie(t))
  - **Scenario Context**: Generation mix configuration (Scenario 1-4)
  
  ### Process Summary
  1. **Initialize** simulation with specific (H‚ÇÅ, H‚ÇÇ) values
  2. **Run** dynamic simulation in MATLAB/Simulink
  3. **Record** Œîf‚ÇÅ, Œîf‚ÇÇ, and ŒîP_tie time series
  4. **Store** the (H‚ÇÅ, H‚ÇÇ) pair with its corresponding responses
  5. **Repeat** 1000 times with different (H‚ÇÅ, H‚ÇÇ) combinations
  6. **Repeat entire process** for all four scenarios
  
  This methodology generates a robust dataset that captures the relationship between inertia constants and system dynamic responses across diverse generation scenarios, providing sufficient data for effective machine learning model training and validation.
  
## Machine Learning Models

  Five machine learning algorithms were implemented and compared:
  1. Random Forest - Ensemble learning method using multiple decision trees
  2. Multiple Linear Regression - Baseline linear model for comparison
  3. XGBoost - Gradient boosting algorithm known for high performance
  4. Support Vector Regression (SVR) - Kernel-based regression technique
  5. Multilayer Perceptron (MLP) - Neural network approach for capturing non-linear relationships

  ### Key Insights
  
  #### Performance Ranking (Average Across All Scenarios)
  1. **Multilayer Perceptron (MLP)** - Best overall (2.14-4.67% MAPE)
  2. **Random Forest** - Strong competitor, especially for H‚ÇÅ (1.88-8.80% MAPE)
  3. **XGBoost** - Consistent performance across scenarios (2.16-8.83% MAPE)
  4. **Support Vector Regression (SVR)** - Good for complex scenarios (3.01-7.96% MAPE)
  5. **Multiple Linear Regression** - Baseline method, less accurate for non-linear relationships
  
  #### Scenario Difficulty Analysis
  - **Easiest**: Scenario 2 (Hydro-dominated) - All algorithms achieved < 5.39% MAPE
  - **Most Consistent**: Scenario 3 (Symmetric) - Balanced performance across both areas
  - **Most Challenging**: Scenario 4 (Three-area) - Complex interactions but MLP maintained < 4% MAPE
  
  #### Algorithm Strengths
  - **MLP**: Excels at capturing non-linear relationships in complex systems
  - **Random Forest**: Robust to overfitting, good for smaller datasets
  - **XGBoost**: Efficient handling of missing data and feature importance
  - **SVR**: Effective in high-dimensional spaces with clear margin of separation
  
  These results demonstrate that machine learning, particularly MLP, provides highly accurate inertia estimation (< 5% error) across diverse power system configurations, offering a practical solution for real-time grid monitoring.
